# e2eテストルール

## E2E テスト（End-to-End Test）

- 目的: ユーザーが想定通りにアプリを操作できることを検証する
- 対象: ユーザーフロー全体（例: ログイン → 一覧 → 詳細ページ）
- 特徴:
  - ブラウザ上で実行（Playwright, Cypress など）
- 技術例: Playwright
- 配置場所例: プロジェクトルートの `__e2e__` ディレクトリ
